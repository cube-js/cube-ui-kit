### ItemButton

Combines actionable behavior via the `useAction` hook with the visual layout and theming of `ItemBase`. Use it for list-like buttons with optional icons, prefixes/suffixes, and descriptions.

### Base Properties

Supports [Base properties](/BaseProperties)

### Styling Properties

#### styles

Customizes the root `ItemBase` appearance.

Sub-elements:
- `Icon` - left icon wrapper
- `RightIcon` - right icon wrapper
- `Content` - main text container
- `Description` - secondary text line
- `Prefix` - content before the main text
- `Suffix` - content after the main text

### Style Properties

Supports container and text style props (e.g., `padding`, `radius`, `fill`, `color`, `width`, `height`, `flow`, `gap`).

### Modifiers

You can override state via `mods`.

- `hovered`/`pressed`/`focused`/`disabled` - interaction states from `useAction`
- `with-icon`/`with-right-icon`/`with-prefix`/`with-suffix`/`with-description` - layout states from `ItemBase`

### Variants

Types: `primary`, `secondary`, `outline`, `neutral`, `clear`

Themes: `default`, `danger`, `success`, `special`

Sizes: `xsmall`, `small`, `medium`, `large`, `xlarge`, `inline`

### Accessibility

- Keyboard: Tab to focus, Enter/Space to activate
- Screen readers: Provide `label` or visible children
- ARIA: pass `aria-*` props as needed

### Best Practices

1. Prefer visible text or set `label` for icon-only cases
2. Use `to` for navigation; supports `!` for new tab and `@` for native navigation
